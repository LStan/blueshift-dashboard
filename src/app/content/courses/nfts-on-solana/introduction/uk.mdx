# NFT на Solana

![NFT на Solana](/graphics/course-banners/nfts-on-solana.png)

Невзаємозамінні токени (NFT) — це цифрові токени з особливими характеристиками, які відрізняють їх від взаємозамінних токенів, таких як криптовалюти.

На відміну від взаємозамінних токенів, де кожна одиниця ідентична та взаємозамінна, NFT представляють унікальні, неподільні цифрові активи, які неможливо відтворити або розділити.

<ArticleSection name="Властивості NFT" id="nft-properties" level="h2" />

Щоб токен кваліфікувався як справжній NFT, він повинен мати такі основні властивості:
- Неподільність: Не містить десяткових знаків, що гарантує неможливість володіння частками, наприклад, 0,1 NFT
- Унікальність: Має запас рівно 1, що гарантує існування лише одного екземпляра
- Фіксований запас: Відсутність права на карбування, що запобігає створенню додаткових токенів
- Групованість: Можливість організації в колекції для групування подібних токенів, зберігаючи при цьому індивідуальну унікальність
- Багаті метадані: Включає читабельні дані, такі як назви, описи, зображення та інші атрибути для відображення в оглядачах і гаманцях

З цієї причини створення NFT передбачає розгортання смарт-контракту або інтерфейсу, який:
- Створює рахунок карбування з нульовою кількістю десяткових знаків
- Карбує єдиний токен у гаманець власника
- Відкликає право на карбування, щоб запобігти майбутньому карбуванню
- Пов'язує облікові записи метаданих, що містять описову інформацію токена

Встановлені протоколи значно спрощують цю складність. Розробники Solana зазвичай використовують програми, такі як `mpl-token-metadata` або `mpl-core`, розроблені [Metaplex](https://metaplex.com), які обробляють технічні деталі реалізації та надають стандартизовані інтерфейси для створення та управління NFT.

<ArticleSection name="Чому невзаємозамінні токени важливі" id="why-non-fungible-tokens-matter" level="h2" />

NFT представляють перше технічне рішення, що забезпечує доказове цифрове володіння без довірених посередників. До появи NFT цифрові активи існували виключно в межах централізованих платформ, де право власності могло бути довільно скасоване.

> NFT встановлюють право власності, яке функціонує незалежно на рівні протоколу, створюючи справжні цифрові майнові права.

### Переваги перед фізичними активами

NFT покращують володіння фізичними активами, забезпечуючи програмовану логіку, яка реагує на заздалегідь визначені умови.

Смарт-контракти можуть автоматично розподіляти роялті творцям щоразу, коли їхні NFT продаються на вторинних ринках.

Обмеження на передачу можуть запобігти несанкціонованому переміщенню чутливих активів, тоді як часові механізми можуть розблоковувати функції або змінювати властивості на основі конкретних дат чи умов.

Деякі NFT навіть динамічно оновлюють свої візуальні або функціональні властивості, реагуючи на зовнішні джерела даних або взаємодію з користувачами.

### Нові економічні моделі

NFT відкривають економічні структури, які не можуть підтримувати традиційні системи.

Економіка творців трансформує спосіб отримання доходу митцями, встановлюючи постійні потоки роялті — щоразу, коли NFT змінює власника, оригінальний творець автоматично отримує відсоток від ціни продажу.

Це створює стійкі моделі доходу, які виходять далеко за межі початкових продажів.

Спільне володіння виникає через колективне управління NFT, де групи можуть спільно володіти цінними цифровими активами та приймати рішення щодо їх використання через децентралізовані механізми голосування.

Тим часом, компенсація цифрової праці забезпечує нові рамки для винагородження внесків в онлайн-спільноти, творчі співпраці та розробку віртуальних світів способами, які не можуть забезпечити традиційні структури зайнятості.

### Компонованість та інтероперабельність

NFT досягають повної компонованості між додатками та протоколами, створюючи безпрецедентну гнучкість у корисності цифрових активів.

Один ігровий предмет NFT може одночасно функціонувати в кількох іграх з різними механіками, відображатися в цифрових галереях та віртуальних світах, служити заставою в DeFi-протоколах для позик або фармінгу доходності, а також взаємодіяти з іншими протоколами через інтеграції смарт-контрактів.

Ця інтероперабельність працює без необхідності отримання дозволу від оригінального творця, встановлюючи справді відкриті цифрові економіки, де активи набувають цінності через мережеві ефекти та міжплатформну корисність, а не через обмеження, специфічні для платформи.

<ArticleSection name="Collections" id="collections" level="h2" />

Колекція представляє групу NFT, які мають спільні атрибути, теми або походження, створюючи перевірені онлайн-зв'язки між різними активами.

Оскільки теоретично будь-хто міг би створити NFT і неправдиво заявити про належність до престижної колекції, колекції мають авторитет (зазвичай це оригінальний творець або уповноважена особа), який може перевірити, які NFT законно належать до колекції.

Цей процес перевірки створює незмінний онлайн-запис, що зберігається в метаданих NFT, встановлюючи автентичне членство в колекції, яке неможливо підробити або маніпулювати після створення.

Колекції виконують кілька критичних функцій в екосистемі NFT. Вони забезпечують ефективне виявлення та перегляд пов'язаних активів, надають ідентичність бренду та тематичну узгодженість для творців, встановлюють перевірку походження та автентичності, а також створюють основи для спільного управління та спільного досвіду серед власників активів колекції.

Крім того, вони створюють дуже ефективний спосіб логічного групування пов'язаних токенів, зберігаючи при цьому їхню індивідуальну унікальність і властивості, що робить контроль доступу для однотипних активів набагато ефективнішим і дешевшим.

<ArticleSection name="Metadata" id="metadata" level="h2" />

Метадані слугують описовим шаром, який перетворює необроблені токени NFT на значущі цифрові активи.

У той час як блокчейн зберігає лише основну інформацію про токен: власність, пропозицію та базові ідентифікатори; метадані надають багатий контекст, який робить взаємозамінні та невзаємозамінні токени цінними та придатними для використання в додатках.

Структура метаданих зазвичай відповідає встановленим стандартам, щоб гаманці, маркетплейси та додатки могли послідовно інтерпретувати та відображати інформацію NFT незалежно від того, яка програма створила токен.

Метадані включають фундаментальні елементи, такі як назва та опис активу, які надають зрозумілий для людини контекст того, що представляє NFT.

Поле зображення містить URI, що вказує на візуальний актив, будь то цифрове мистецтво, зображення профілю чи ілюстрація ігрового предмета. Для складніших медіа додаткові поля можуть посилатися на анімації, 3D-моделі або інтерактивний контент, який оживляє NFT у сумісних додатках.

Користувацькі атрибути розширюють метадані за межі базових описів, забезпечуючи складні системи класифікації та фільтрації. Ці атрибути можуть представляти риси рідкісності в колекційних предметах, статистичні властивості в ігрових активах або параметри корисності у функціональних токенах. Гнучка система атрибутів дозволяє творцям вбудовувати будь-які релевантні дані, що підвищують цінність або функціональність NFT.

Деякі проєкти зберігають метадані безпосередньо в блокчейні для максимальної довговічності (за допомогою процесу, який ми називаємо інскрипцією), інші використовують децентралізовані рішення для зберігання, такі як IPFS або Arweave, щоб збалансувати вартість із довговічністю, деякі використовують централізовану базу даних. Основна відмінність полягає в тому, що обраний метод зберігання безпосередньо впливає на довгострокову доступність NFT та поточні обов'язки творця щодо підтримки пов'язаного контенту.
